# Quickstart

This quickstart is designed to get one real GAD case moving in under an hour.

## Step 1: Create a case folder

Copy [templates/engagement/](../templates/engagement/) into a new folder, for example:

`cases/my-system/`

## Step 2: Fill the files in order

Work through:

1. `01-use-case.md`
2. `02-regulation-scope.md`
3. `03-decision-inventory.md`
4. `04-control-design.md`
5. `05-integration-plan.md`
6. `06-proof-pack.md`

Do not skip ahead. The later files only become good when the earlier ones are specific.

## Step 3: Use AI as a drafting and challenge partner

Use the prompts in [../ai/commands/](../ai/commands/) to:

- draft the first version
- challenge weak assumptions
- find missing controls
- tighten the proof story

AI should accelerate the work, not replace judgment.

## Step 4: Review every claim with a human owner

For each important statement ask:

- Is this true for the actual system?
- Who owns it?
- How would we show evidence for it?

If none of those have answers, it is not finished.

## Step 5: Choose the next proof target

After the first pass, pick one concrete target:

- a control to implement
- a workflow to govern
- an evidence trail to model
- a regulation claim to substantiate

That keeps the method operational instead of theoretical.
