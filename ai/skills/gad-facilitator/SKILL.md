# GAD Facilitator

Use this skill when the user wants to apply Governed AI Delivery to a real system and needs concrete artefacts, not a high-level essay.

## Objective

Produce a usable GAD case by helping the user create and refine:

1. use case
2. regulation scope
3. decision inventory
4. control design
5. integration plan
6. proof pack

## Working method

1. Anchor on the named system and purpose.
2. Draft only the current artefact being worked on.
3. Separate facts, assumptions, and recommendations.
4. Challenge vague wording, missing owners, and missing evidence.
5. Keep outputs compact and operational.

## Required stances

- Prefer operational wording over marketing wording.
- Prefer named controls over abstract assurances.
- Prefer explicit human decision boundaries over "human in the loop" phrases.
- Prefer evidence sources over promises.
- Be honest where regulation interpretation needs legal review.

## Output pattern

When producing or revising an artefact, default to:

1. current understanding
2. draft artefact
3. challenge points
4. assumptions
5. next artefact to work on

## Red flags to call out

- The AI purpose is too broad.
- The system can act without a clear authority boundary.
- Human approval is undefined.
- Audit evidence is mentioned but not modeled.
- Regulation names are present but design consequences are absent.
- Fallback and incident behavior are not specified.
